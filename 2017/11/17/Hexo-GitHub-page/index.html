搭建Hexo + GitHub page博客总结

发表于 2017-11-17 | 分类于 知识工具总结

  • 创建Github page库

创建Github page库

在github创建一个Github page库。不需要增加branch。

GitHub Pages 是用来存放个人主页的仓库，每个用户只能有一个。仓库的名字必须是username.github.io。你可以通过http://username.github.io 来访问你的个人主页。

安装Hexo和配置Hexo文件

安装和配置Hexo的教程在Hexo官网写得非常详细，里面的视频教程讲得非常好。建议跟着视频走，然后再看帮助文档。

在配置themes的时候建议尽量选择比较多star的theme（这样基本能保证跟得上hexo更新）。 一般来说每个theme都在它们的github repo有说明怎么进行theme配置，这也是要仔细阅读的。这是我采用的next theme。

设置ssh公钥

1.打开terminal，用cd ~/.ssh检查我们电脑上是否有ssh公钥。如果返回No such file or directory或者在~/.ssh这个文件夹里没有id_rsa.pub这个文件夹，我们就需要生成新的SSH公钥了。有公钥但未在github设置的可跳至第三步。若两者都设置了就直接跳到部署博客部分。

2.用$ ssh-keygen -t rsa -C "邮件地址@youremail.com"生成新公钥，@处填写你的邮箱。接着问你放置的文件夹时，用默认文件夹就好（直接回车）。接着会让你设置密码及确定密码。注意密码必须5位或以上。

3.接着你就可以进入~/.ssh这个文件夹，打开id_rsa.pub这个文档。将里面的一串公钥复制出来。然后去你的github-Account Settings—>SSH Public keys —> add another public keys，然后粘贴进去确认就行了。

4.用ssh -T git@GitHub.com查看是否设置成功。如返回

The authenticity of host ‘GitHub.com (207.97.227.239)’ can’t be established. RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

这时输入yes就好。（若对其中原理感兴趣的可以去看看SSH原理与运用这篇博客）

接着如果我们看到

那就是设置成功了。

部署博客到github

1.进入博客文件夹安装一个远程部署的扩展
$ npm install hexo-deployer-git --save

2.在config文档的deploy属性里加入你的github page的地址

3.最后部署到远端
$ hexo generate --deploy
